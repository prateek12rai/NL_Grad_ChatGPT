# Phase 1 — Foundation (Plain English)

## What this phase is about

Think of this as **building the empty house** before moving in furniture.

We are **not** downloading medical papers yet and **not** answering questions yet. We are setting up:

- Folders and basic project files so everything has a proper place
- A tiny “health check” backend so we can confirm the system is alive
- A simple Streamlit page (for later deployment) that says the app works
- A simple website shell (for later Vercel) that can build successfully
- Templates showing which **secret keys** you will need later (Groq, Hugging Face) — without putting real keys in the project
- A **privacy filter** that blocks Indian ID numbers and personal details from being stored
- Automated tests so each future phase can be checked safely

## What you will see when it is done

- A `PHASES/Phase-01-Foundation/GATE_REPORT.md` file saying **PASS** or **FAIL**
- Project folders like `src/`, `backend/`, `frontend/`, `tests/`
- You do **not** need to run anything yourself unless you want to — we run the deep test and record results

## What we will NOT do in Phase 1

- No scraping government or Nature websites
- No AI answers to medical questions
- No deployment to the internet yet (only local “does it start?” checks)

## Time idea

Usually **1–2 sessions** of setup and testing.
